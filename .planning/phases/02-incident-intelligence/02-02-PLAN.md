**Function 1: `GroupAndCreateIncidents(db *sql.DB) (int, int, error)`**
- Returns: (events_processed, incidents_created, error)
- Step 1: Query unprocessed events (those NOT in incident_events, with non-empty source_ip)
- Step 2: Sort by source_ip ASC, timestamp ASC (already sorted by SQL ORDER BY)
- Step 3: Group by source_ip within 15-minute windows using the sliding window algorithm
- Step 4: For each group, create an Incident record with aggregated metadata
- Step 5: Persist each incident via database.CreateIncident
- Step 6: Seed the techniques table with AllTechniques() if empty

**Grouping algorithm** (Go-side after SQL fetch — research showed SQL window functions are ideal but the current SQLite setup with 442K events is manageable in Go with proper sorting):
```go
func GroupEventsIntoIncidents(events []database.Event, windowMinutes int) [][]database.Event {
    if len(events) == 0 {
        return nil
    }
    
    var groups [][]database.Event
    currentGroup := []database.Event{events[0]}
    
    for i := 1; i < len(events); i++ {
        prev := events[i-1]
        curr := events[i]
        
        // New group if source IP changes
        if curr.SourceIP != prev.SourceIP {
            groups = append(groups, currentGroup)
            currentGroup = []database.Event{curr}
            continue
        }
        
        // New group if time gap > window
        prevTime, _ := time.Parse(time.RFC3339, prev.Timestamp)
        currTime, _ := time.Parse(time.RFC3339, curr.Timestamp)
        gap := currTime.Sub(prevTime)
        
        if gap > time.Duration(windowMinutes)*time.Minute {
            groups = append(groups, currentGroup)
            currentGroup = []database.Event{curr}
        } else {
            currentGroup = append(currentGroup, curr)
        }
    }
    
    groups = append(groups, currentGroup)
    return groups
}
```

**Function 2: `BuildIncidentFromGroup(events []database.Event) database.Incident`**
- Set SourceIP from first event
- Set StartTime from first event, EndTime from last event
- Set EventCount = len(events)
- Collect unique users, IPs, hostnames into maps, then convert to sorted slices
- Map each event to ATT&CK techniques via `attck.MapEventToTechniques`
- Deduplicate techniques by ID, count events per technique
- Collect unique tactics from techniques
- Calculate severity via `CalculateSeverity`
- Generate title: "{Primary Technique Name} from {Source IP}" or "Security Incident from {Source IP}" if no techniques
- Calculate confidence: min(1.0, float64(eventCount) / 50.0) — more events = higher confidence, capped at 1.0

**Function 3: `CalculateSeverity(eventCount int, eventTypes []string, techniques []string) string`**
- Event type weights: privilege_escalation=10, authentication=3, network_activity=5, file_activity=4, file_create=4, file_delete=6, process_activity=5, registry_access=7, database_query=4, system=1, ntlm_auth_success=4, service_start=2, scheduled_task=3
- Technique severity weights: T1110=8, T1021=9, T1548=9, T1059=6, T1053=7, T1550=7, T1558=8, T1070=5, T1112=4, T1562=7, T1003=9, T1087=3, T1005=4, T1048=8
- Sum all weights, normalize: >50 → "critical", >30 → "high", >15 → "medium", else "low"
