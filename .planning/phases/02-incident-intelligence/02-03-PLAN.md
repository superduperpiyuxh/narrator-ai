The handler struct should embed the existing Handler pattern — access db via the existing Handler's db field. Create a new `IncidentHandler` that wraps the base handler:

```go
package handler

import (
    "database/sql"
    "net/http"
    "strconv"
    
    "github.com/gin-gonic/gin"
    "github.com/superduperpiyuxh/narrator-ai/backend/internal/attck"
    "github.com/superduperpiyuxh/narrator-ai/backend/internal/database"
    "github.com/superduperpiyuxh/narrator-ai/backend/internal/incident"
)

type IncidentHandler struct {
    db *database.DB
}

func NewIncidentHandler(db *database.DB) *IncidentHandler {
    return &IncidentHandler{db: db}
}
```

**Endpoint 1: POST /api/incidents/group**
- Handler: `GroupIncidents(c *gin.Context)`
- Calls `incident.GroupAndCreateIncidents(h.db.Conn())`
- Returns: `{"grouped": N, "incidents_created": N, "duration": "2.3s"}`
- Measure duration with `time.Now()` / `time.Since()`
- Note: need to expose `Conn()` method on database.DB (add `func (db *DB) Conn() *sql.DB { return db.conn }` to sqlite.go)

**Endpoint 2: GET /api/incidents**
- Handler: `GetIncidents(c *gin.Context)`
- Query params: limit (default 50), offset (default 0), severity, status, source_ip
- Calls `database.GetIncidents(h.db.Conn(), limit, offset, severity, status, sourceIP)`
- Returns: `{"incidents": [...], "total": N, "limit": N, "offset": N}`

**Endpoint 3: GET /api/incidents/:id**
- Handler: `GetIncident(c *gin.Context)`
- Parses :id as int64
- Calls `database.GetIncidentByID(h.db.Conn(), id)`
- Returns 404 if nil: `{"error": "incident not found"}`
- Returns: `{"incident": {...}}`

**Endpoint 4: GET /api/incidents/:id/events**
- Handler: `GetIncidentEvents(c *gin.Context)`
- Parses :id as int64
- First verify incident exists via GetIncidentByID
- Calls `database.GetIncidentEvents(h.db.Conn(), id)`
- Returns: `{"events": [...], "total": N, "incident_id": N}`

**Endpoint 5: GET /api/incidents/stats**
- Handler: `GetIncidentStats(c *gin.Context)`
- Calls `database.GetIncidentStats(h.db.Conn())`
- Returns the stats map directly

**Endpoint 6: GET /api/techniques**
- Handler: `GetTechniques(c *gin.Context)`
- Returns `attck.AllTechniques()` as JSON
- Returns: `{"techniques": [...]}`

Implementation notes:
- Follow the same error response pattern as existing handler.go: `c.JSON(http.StatusInternalServerError, gin.H{"error": err.Error()})`
- Validate path parameters with strconv.ParseInt, return 400 on parse failure
- The `Conn()` method must be added to database.DB in sqlite.go (one-line accessor)

## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| HTTP → API handler | User-controlled query params (limit, offset, severity) must be validated |
| API handler → database | Parameterized queries prevent SQL injection |
| Grouping endpoint → system | POST trigger can be called repeatedly — must be idempotent |

| T-02-06 | Information Disclosure | GET /api/incidents returns all incidents | accept | Hackathon demo, no auth needed; pagination limits response size |
| T-02-07 | Tampering | Query param injection (severity, status) | mitigate | Parameterized queries; whitelist valid values |
