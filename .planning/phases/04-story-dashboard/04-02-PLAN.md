   - Fetch incident: `fetchIncident(Number(id))` from @/lib/api
   - Fetch narrative: `fetchNarrative(Number(id))` from @/lib/api — catch 404 (no narrative yet)
   - Layout:
     - Back button: Link to "/" with arrow-left icon, "Back to Incidents"
     - Header: incident.title + SeverityBadge + incident.id monospace
     - Metadata row: source IP (monospace), time range (start_time — end_time), event count, unique users/IPs counts
     - MITRE techniques: full list of TechniqueBadge components with event counts
     - StoryCard component: pass narrative data (if exists) or "No narrative generated" state with button to trigger generation (links to POST endpoint hint)
   - If incident not found: render 404 state with "Incident not found" message and back link

2. Create `frontend/src/components/StoryCard.tsx` as a Client Component ('use client'):
   - Props: `narrative: Narrative`, `incidentId: number`
   - Parse narrative.sentences from JSON string to Sentence[] array
   - State: `hoveredSentenceIndex: number | null`, `selectedEventIds: number[]`
   - Layout: bg-zinc-900 border border-zinc-800 rounded-xl overflow-hidden
   - Header: "Attack Narrative" title + ConfidenceBadge for overall narrative confidence + model used badge
   - Summary section: narrative.summary in italic text-zinc-300
   - Sentences section: map over parsed sentences, render NarrativeSentence for each
   - On sentence hover: set hoveredSentenceIndex, set selectedEventIds to sentence.source_event_ids
   - On sentence leave: clear hoveredSentenceIndex and selectedEventIds after 200ms delay (prevents flicker when moving between sentences)
   - Metadata footer: tokens used, generation time, model temperature — displayed in monospace text-xs text-zinc-500

3. Create `frontend/src/components/NarrativeSentence.tsx` as a Client Component:
   - Props: `sentence: Sentence`, `index: number`, `isHovered: boolean`, `onHover: (eventIds: number[]) => void`, `onLeave: () => void`
   - Render: p-3 rounded-lg cursor-pointer transition-all duration-150
   - Normal state: text-zinc-200, hover:bg-zinc-800/50
   - Hovered state: bg-zinc-800 border-l-2 border-blue-500, text-zinc-50
   - Sentence text: sentence.text
   - Below text: flex row with timestamp (text-xs text-zinc-500), TechniqueBadge if sentence.technique exists, ConfidenceBadge for sentence.confidence
   - onMouseEnter: call onHover(sentence.source_event_ids)
   - onMouseLeave: call onLeave()

4. Create `frontend/src/components/RawEventViewer.tsx` as a Client Component:
   - Props: `eventIds: number[]`, `narrativeId: number`
   - Use SWR to fetch events: `useSWR(eventIds.length > 0 ? `/api/narratives/${narrativeId}` : null, fetcher)`
   - When eventIds change, filter the fetched events to only show those in eventIds
   - If no eventIds selected: show placeholder "Hover a sentence to view source events" with crosshair icon
   - If loading: show skeleton with 3 pulse bars
   - If error: show "Failed to load source events" in text-red-400
   - If events found: render each event as a card:
     - bg-zinc-950 border border-zinc-800 rounded-lg p-3 mb-2
     - Header row: event.id monospace badge + event.event_type badge (colored by type)
     - Details: timestamp, hostname, source_ip, dest_ip in monospace text-xs
     - If event has process_name or command_line: show in monospace text-xs text-zinc-400 with "Process:" and "Command:" labels
     - Expandable: click to show full raw_json in collapsible section (pre-formatted, text-xs)
   - Layout: sticky top-4, max-h-[calc(100vh-8rem)] overflow-y-auto

5. Create `frontend/src/hooks/useFeedback.ts`:
   - Custom hook using SWR for feedback data + mutation
   - `useFeedback(narrativeId: number)` returns:
     - `feedback: Feedback | null` — existing feedback (or null)
     - `isSubmitting: boolean`
     - `submitFeedback(rating: 'up'|'down', notes: string)` — calls POST /api/feedback, mutates SWR cache
     - `hasSubmitted: boolean` — true if feedback already exists for this narrative
