1. Add feedback table migration to `backend/internal/database/sqlite.go` in the migrate() function:
   ```sql
   CREATE TABLE IF NOT EXISTS feedback (
       id INTEGER PRIMARY KEY AUTOINCREMENT,
       narrative_id INTEGER NOT NULL,
       incident_id INTEGER NOT NULL,
       rating TEXT NOT NULL CHECK (rating IN ('up', 'down')),
       notes TEXT DEFAULT '',
       user_id TEXT,
       created_at TEXT DEFAULT (datetime('now')),
       FOREIGN KEY (narrative_id) REFERENCES narratives(id) ON DELETE CASCADE,
       FOREIGN KEY (incident_id) REFERENCES incidents(id) ON DELETE CASCADE
   );
   CREATE INDEX IF NOT EXISTS idx_feedback_narrative ON feedback(narrative_id);
   CREATE INDEX IF NOT EXISTS idx_feedback_incident ON feedback(incident_id);
   ```
   Place this after the narratives table creation. Use user_id TEXT with NULL default for forward compatibility with future auth.

2. Create `backend/internal/database/feedback.go`:
   - `Feedback` struct matching the table schema (ID, NarrativeID, IncidentID, Rating, Notes, UserID, CreatedAt)
   - `CreateFeedback(f *Feedback) error` — INSERT and return with generated ID
   - `GetFeedbackByNarrativeID(narrativeID int64) (*Feedback, error)` — SELECT single feedback record
   - `GetFeedbackByIncidentID(incidentID int64) ([]Feedback, error)` — SELECT all feedback for an incident
   - Use parameterized queries (no string interpolation) for SQL injection prevention

3. Create `backend/internal/handler/feedback_handler.go`:
   - `FeedbackHandler` struct with db field
   - `NewFeedbackHandler(db *database.DB) *FeedbackHandler`
   - `SubmitFeedback(c *gin.Context)` — POST handler:
     - Parse JSON body: narrative_id (required), incident_id (required), rating ("up"|"down", required), notes (optional string, max 1000 chars)
     - Validate narrative_id and incident_id are positive integers
     - Validate rating is "up" or "down"
     - Sanitize notes: trim whitespace, enforce max length
     - Call db.CreateFeedback()
     - Return 201 with created feedback object
   - `GetFeedback(c *gin.Context)` — GET handler:
     - Parse narrative_id from URL param
     - Call db.GetFeedbackByNarrativeID()
     - Return feedback object or 404 if none exists

4. Register routes in `backend/main.go`:
   - Create feedback handler: `fh := handler.NewFeedbackHandler(db)`
   - Add routes after narrative routes:
     - `r.POST("/api/feedback", fh.SubmitFeedback)`
     - `r.GET("/api/feedback/:narrative_id", fh.GetFeedback)`
